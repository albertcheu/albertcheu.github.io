NetVis

created by Albert Cheu, Hira Javaid, and Supraja Krishnan

Motivation

Net neutrality is the practice of providing a uniform level of access to web content; it is violated when an internet service provider (ISP) gives high speed to one application and low speed to another. Researchers call such violations "differentiation." In the United States, the Open Internet Order once protected net neutrality, but the Order's elimination by the FCC in late 2017 has left the future in question.

From a market perspective, consumers could make informed decisions if the scale of differentiation was public knowledge. And from a societal perspective, citizens and lawmakers could use the data to strengthen their arguments about net neutrality.

Data

To achieve this goal, a team at Northeastern University created Wehe, a mobile app that tests if a user is experiencing differentiation. This site provides a visual tool to explore several months worth of Wehe tests.

The data contains tests of Amazon, NBCSports, Netflix, Skype, Spotify, Vimeo, and Youtube usage. Although a plurality of the tests were conducted in the United States, many countries are represented.

Task Analysis

We interviewed one of Wehe's developers to pin down the necessary tasks. We discovered that the desired functionalities were progressive refinements of a comparison query. For a particular country in a specified window of time, one ought to be able to compare the number of positive and negative results for a selection of ISPs.

Design Process

The tasks being primarily filters, we first set about designing the most clear way of enabling them. As geography plays a role in the filtering, a map is the most natural choice. Due the large number of tests in the United States, we also decided to have a US map to see breakdown by state. The most intuitive way to filter by time is through a brush; our design also has a focus view that is evocative of a funnel. The dropdown for ISP was also natural, as we had to allow users to select multiple discrete objects.

Because the core task is comparison between quantitative values, the lengths in a bar chart are both effective and expressive. We chose red to indicate differentiation as it pops-out, esp. in contrast to black.

Final Visualization

The bulk of NetVis is made with the Javascript library D3. The select2 jQuery package enables multi-select. Map data is drawn from json files hosted here and here. The SweetAlert package is used to show alerts as popups, in case an ISP and country aren't selected.

Data Analysis

Missing values were abundant: 0,0 latitude and longitude and empty ISP names required processing. Noisy data like multiple ways of spelling a single ISP was also a problem.

As stated previously, many of the tests were conducted in the US, which forced us to apply log-scaling to the choropleth. We also noted a peak in the number of tests in late January, due to media coverage of the Wehe project.

Conclusion

NetVis achieves the core requirement of a detailed ISP comparison as well as a time- and place-based overview. Further work will add a city-level filtering.
